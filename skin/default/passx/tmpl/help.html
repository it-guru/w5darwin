Password Exchange System - PassX

Jeder Administrator kennt das Problem der zyklischen Passwort Änderung auf den verschiedensten Systemen. An der Tatsache das vor allem die Root-Kennung zyklische neue Passworte benötigen, kann PassX nichts ändern. PassX kann aber dabei unterstützen, diese Änderungen automatisiert durchzuführen, und die neu gesetzten Passworte dann den jeweiligen Administratorn und Nutzern gezielt auf einem sicheren Weg verfügbar zu machen.

PassX Evolution II

PassX ist ein Konzept zur automatischen Änderung und Verteilung von Passworten. Dies kann z.B. für root Kennungen oder andere Systemkennungen verwendet werden, bei denen ein Passwort zwingend notwendig ist.
Alle Passwort Informationen werden verschlüsselt abgelegt, und einem genau definierten Benutzerkreis zugäglich gemacht. Dieser Schutz ist so effizient, das selbst der Administrator des Servers, auf dem die Passwortinformationen gespeichert sind, keinen Zugang zu diesen Daten hat.

PassX erfüllt die höchsten Sicherheitsanforderungen an den Passwortschutz. Es werden 8 stellige Passworte generiert, die Groß-/ Kleinschreibung und numerische Werte kombinieren. Der Änderungszyklus kann beliebig gewählt werden. So sind täglich, wöchentlich oder monatlich die am einfachsten zu konfigurierenden Zyklen - beliebige andere Zyklen aber jederzeit durch den Administrator des jeweigen Servers einstellbar.
Die Übertragung der Passworte über das TCP/IP Netz an einen zentralen Punkt (PassX-Server) erfolgt RSA-Verschlüsselt. Ein "Mitlauschen" oder "Abhören" der IP-Verbindung ist somit zwecklos. Die Abfrage der Passworte vom zentralen PassX Server ist nur über einen JavaScript-WEB-Browser möglich, auf dem im Browser die entschlüsselung der Passwort-Daten statfindet. Netztechnischen "Lauschangriffen" wird somit keine Angriffsfläche geboten.
Der gesammte Prozess basiert auf OpenSource Software, und ist somit einem Maximum an Transparenz unterworfen. Durch die Offenlegung der Source-Codes ist eine eingehende Prüfung auf Sicherheitsprobleme durch jeden Programmiere/ Softwarespezialisten möglich. Dieses Level an Sicherheit kann durch keine rein kommerziell entwickelte Software gehalten werden. Das keinerlei Lizenskosten entstehen, ist dabei ein zusätzlicher angenehmer Nebeneffekt.

Per Commandline Passworte ändern und in PassX abspeichern
Mit dem Commandline Tool "ChangePassword" können (aktuell nur auf Unix-Systemen) Passworte geändert und zum PassX Server übertagen werden. "ChangePassword" kennt div. Optionen, die mit "ChangePassword --help" abrufbar sind. Die einfachste Variaten ist z.B. das zyklische Ändern des root Passwortes mit folgendem cron Eintrag:
10 * * * * /usr/local/bin/ChangePassword --base http://mypassxserver/w5base/public/
Die grunsätzliche Arbeitsweise von ChangePassword läuft dann wie folgt ab:
  • Es werden die Public-Keys der User vom PassX Server angefordert, die lesenden Zugang auf die jeweilige Kennung (in diesem Fall root) erhalten sollen.
  • Es wird ein zufälliges Passwort generiert.
  • Das Passwort wird in die /etc/passwd bzw. /etc/shadow eingetragen
  • Für die Lister der Public-Keys wird das Passwort mittels OpenSSL verschlüsselt und in verschlüsselter Form an den PassX Server übermittelt (Web-Request).
Da ChangePassword auch die Möglichkeit alternativer PasswordChanger bietet, kann z.B. auch das Passwort in einer Anwendung geändert werden. Aktuell stehen folgende PasswordChanger zur Verfügung: 010Shadow, 020Passwd, oracle und storeonly

Mit ChangePassword Passworte ändern, die als Privat gekennzeichnet sind
Beim Kommando ChangePassword die Option --entrytypeid 11 mitgeben!

Wie kann ich eine Liste von Passworten per Script in PassX laden?
Dies ist nicht über den Web-Browser möglich, da der Upload von Listen über Excel erfoglt, und Excel die Daten nicht verschlüsseln kann. Es ist aber möglich, diese Listen mit dem ChangePassword Tool in PassX zu laden. z.B.:
ChangePassword --changer storeonly --host myhost --user root --base http://myw5baseserver/w5base/public/ --pass 1234
