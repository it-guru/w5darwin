W5Base/Darwin

Data Access and Research Web-Interface

Darwin - so siehts aus

Darwin - wozu und warum

  • SACM Prozesstool - Configuration-Management
  • Verfahren zur Datenqualitätssicherung (DataIssue Verfahren) und Datenvalidierung beim Verädern von Daten
  • Workbench für den Zugriff auf die verschiedensten Unternehmensdaten (alle Daten für die tägliche Arbeit in EINER Oberfläche)
  • Workflow Engine
  • div. kleinere Datentools (PassX Passwort-Verwaltung, FAQ-Datenbank, Forum-System, ...)

Darwin - die Technik

  • Perl "from scratch", Apache Web-Server, MySQL Datenbank
  • Basierend auf W5Base Framework (http://sf.net/p/w5base)
  • Entwickelt zu 100% auf OpenSource Basis (etwa 70 CPAN Bibliotheken)
  • Designed für JEDEN aktuellen Browser (und einige uralte der Telekom)
  • Unterstützung für Deutsch und Englisch
  • div. API Zugrifsmöglichkeiten (W5API (Perl), J4Base (Javascript) und nativ XML oder SOAP)
Entwicklungstools: (keep it simple)
  • Vim
  • Subversion
  • sqlrunner bzw. OracleDeveloper
